- (매우 심각한 버그) 메모 우클릭 팝업에 글씨가 나타나지 않아서 버튼만 있고 무슨 기능인지 알 수가 없음.

- (기능 변경 사항) 메모 우클릭에서 메모 프리셋을 변경할 수 있음. 프리셋은 3가지 색상의 조합으로 이루어짐

1. 테두리 색상 (border)
2. 텍스트 색상 (text) : 스크롤의 막대 색상이기도 함
3. 메모 영역 전체 색상 (area) : (스크롤 막대의 빈 영역 이기도 함)

그리고 기본적으로 제공되는 프리셋은

1. Snow : White-White-Transparent
2. Ink : Black-Black-Transparent
3. Sky : White-White-Blue
4. Banana : Black-Black-Yellow

- (버그 및 매우 자세한 변경사항) 일반 바탕 외부 우클릭 팝업에 글씨가 나타나지 않음. 그리고 테마로 추정되는 가운데 버튼을 누르더라도 아무 일도 일어나지 않음.

- 일반 바탕 우클릭 팝업을 통한 테마 변경은 컬러 프리셋을 변경하는 것임.

1. 배경 색상 (background) : background 색상은 메모가 Transparent일때의 색상에 영향을 미침.
2. 텍스트 색상 (indicator) : 팝업박스의 테두리와 글씨 색상

그리고 기본적으로 제공되는 프리셋은

1. Dark Base : Black-White
2. Light Base : White-Black
3. Dark Mode : Transparent-White
4. White Mode : Transparent-Black
   (3,4에서 투명은 프로그램이 칠하지 않고, 터미널의 배경색을 그대로 사용하는 것. 시스템이 다크모드일 것으로 예상하고 White)

- 메모 우클릭의 팝업과 바탕 우클릭의 팝업은 동시에 나타날 수 없음. 한쪽이 발생하면 다른 한쪽은 사라져야함.

- (신규) 메모 우클릭 시에, 최소화 기능이 있음. 이것은 이동을 위한 드래그 맨 윗줄을 더블클릭해도 발생함. 최소화 하게 되면 첫줄만 표기하고, 크기가 3줄로 줄어듦. 따라서 텍스트는 한줄만 표기. 이때는 윗변으로 드래그 이동만 가능하고, 크기조절은 불가능하게 됨. 텍스트 편집도 불가능함.

- .stkrc에서 맨 윗줄과 좌,우,아래의 선 세팅을 변경할 수 있음.

* 선 변경 대상

1. 윗변 (양끝 포함)
2. 윗변 제외 나머지 좌,우,아래

- 선 종류
  single (default): Standard lines.
  double: Thicker, two-line border.
  heavy: Bold, single-line border.
  simple: Basic ASCII (+, -, |).

- 스크롤 바가 생기게 되면 본문과 한칸 정도의 여유 공간이 있어야 함.

- 스티커의 중앙점은 스크린의 우변과 아랫변을 나갈 수 없음. 스크린 크기가 사용자에 의해 줄어들게 되서, 스티커의 중앙점이 스크린의 우변과 아랫변 바깥으로 나가게 되면, 안전하게 안쪽으로 밀려남. 만지지 못하는 스티커가 발생하지 않도록 함.
